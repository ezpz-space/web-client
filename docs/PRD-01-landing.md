# PRD-01: 랜딩페이지 & 정보 페이지

> **프로젝트**: ezpz - 창호 견적 비교 웹서비스
> **버전**: 1.0
> **최종 수정**: 2026-02-10

## 1. 개요

이 문서는 ezpz 서비스의 **랜딩페이지**, **햄버거 메뉴(Drawer)**, **도움말**, **고객센터** 페이지의 기능 요구사항과 UI 명세를 정의합니다.

### 대상 라우트

| 라우트 | 페이지 | 설명 |
|--------|--------|------|
| `/` | 랜딩페이지 | 서비스 소개 및 견적 시작 CTA |
| — (Drawer) | 햄버거 메뉴 | 전역 네비게이션 |
| `/help` | 도움말 목록 | 창호 관련 가이드 목록 |
| `/help/:id` | 도움말 상세 | 개별 가이드 콘텐츠 |
| `/contact` | 고객센터 | 문의 폼 |

## 2. 랜딩페이지 (`/`)

### 2.1 Header

| 요소 | 설명 |
|------|------|
| 로고 | 좌측 배치. 클릭 시 `/`로 이동 |
| 햄버거 아이콘 (☰) | 우측 배치. 클릭 시 Drawer 메뉴 오픈 |

- Header는 **sticky** (스크롤 시 상단 고정)
- 배경: 흰색, 하단 1px border (`#E5E5E5`)

### 2.2 Hero 섹션

| 요소 | 상세 |
|------|------|
| Hero 이미지 | 창호가 설치된 인테리어 사진. 전체 너비, 높이 약 60vh |
| 오버레이 텍스트 | 이미지 위 반투명 오버레이 |
| 메인 카피 | **"창호 교체, 어디서부터 시작해야 할지 모르겠다면"** (h1, 흰색, bold) |
| 서브 카피 | "견적부터 확인하세요" (body1, 흰색) |
| CTA 버튼 | **"견적 확인하기"** — Primary Button, `/estimate` 이동 |

**UI 상태:**
- 기본: CTA 버튼 활성화
- 호버/터치: 버튼 색상 약간 어두워짐 (darken 10%)

### 2.3 Feature 섹션

3개의 Feature 카드를 세로(모바일) 또는 가로(데스크톱) 배치합니다.

| # | 아이콘 | 제목 | 설명 (Pain Point → 해결) |
|---|--------|------|--------------------------|
| 1 | 💰 (가격 아이콘) | 얼마나 들지 몰라 불안하다면 | 견적부터 먼저 확인하세요. 예상 비용을 미리 알 수 있습니다 |
| 2 | 🏪 (매장 아이콘) | 여러 매장 돌아다니기 힘들다면 | 한 곳에서 브랜드별 가격을 한눈에 비교하세요 |
| 3 | ✅ (결정 아이콘) | 결정이 너무 복잡하다면 | 쉽고 명확한 숫자로 비교하고, 간편하게 결정하세요 |

**레이아웃:**
- 모바일: 1열, 카드 간격 24px
- 데스크톱 (≥768px): 3열 그리드

### 2.4 실적 섹션

숫자 강조 영역. 카운트업 애니메이션 적용 (viewport 진입 시 트리거).

| 항목 | 수치 | 라벨 |
|------|------|------|
| 누적 견적 | **3,000+** | 누적 견적 건수 |
| 상담 완료 | **2,000+** | 상담 완료 건수 |

**레이아웃:**
- 2열 그리드, 중앙 정렬
- 숫자: h2, bold, Primary Color
- 라벨: body2, Gray

### 2.5 브랜드 섹션

파트너 브랜드 로고를 가로 나열합니다.

| 브랜드 | 로고 |
|--------|------|
| LX 하우시스 | `lx-logo.svg` |
| KCC 글라스 | `kcc-logo.svg` |
| 청암 | `chungam-logo.svg` |

- 로고 높이: 40px, grayscale 기본 → hover 시 컬러
- 중앙 정렬, 로고 간격 32px

### 2.6 FAQ 아코디언 섹션

자주 묻는 질문을 아코디언 UI로 표시합니다.

| # | 질문 | 답변 (요약) |
|---|------|-------------|
| 1 | 견적 금액은 얼마나 정확한가요? | 입력하신 정보를 기반으로 산출한 예상 금액이며, 실제 시공 비용은 현장 확인 후 확정됩니다. |
| 2 | 왜 견적이 범위로 나오나요? | 창호 사양, 부자재, 현장 조건 등에 따라 가격이 달라질 수 있어 최저~최고 범위로 안내합니다. |
| 3 | 실제 업체 견적과 차이가 날 수 있나요? | 네, 현장 상황(철거 난이도, 층수 등)에 따라 차이가 발생할 수 있습니다. 정확한 금액은 방문 견적을 통해 확인하세요. |
| 4 | 단열은 왜 중요한가요? | 창호는 주거 공간의 열손실 중 가장 큰 비중을 차지합니다. 단열 성능이 좋은 창호로 교체하면 냉난방비를 절감할 수 있습니다. |

**UI 동작:**
- 기본: 모든 항목 닫힘
- 클릭: 해당 항목 열림, 다른 항목 자동 닫힘 (단일 오픈)
- 열림 상태: 화살표 아이콘 180도 회전, 답변 영역 슬라이드 다운
- 닫힘 상태: 화살표 아이콘 원위치, 답변 영역 슬라이드 업

## 3. 햄버거 메뉴 (Drawer)

### 3.1 열기/닫기 동작

| 트리거 | 동작 |
|--------|------|
| 햄버거 아이콘 클릭 | Drawer 우측에서 슬라이드-인 |
| X 버튼 클릭 | Drawer 슬라이드-아웃 |
| 배경(Scrim) 클릭 | Drawer 닫힘 |
| ESC 키 | Drawer 닫힘 |

### 3.2 Drawer 콘텐츠

```
┌─────────────────────┐
│  [X 닫기]            │
│                     │
│  홈                 │  → /
│  견적 시작하기       │  → /estimate
│  도움말             │  → /help
│  고객센터           │  → /contact
│                     │
│  ─────────────────  │
│  admin@ezpzspace.com│
└─────────────────────┘
```

- Drawer 너비: 모바일 80vw (max 320px)
- 배경: 흰색
- Scrim: 반투명 검정 (`rgba(0,0,0,0.5)`)
- 메뉴 항목: body1, 패딩 16px, 클릭 영역 전체 너비

### 3.3 접근성

- `role="dialog"`, `aria-modal="true"`
- 열림 시 포커스 트랩 (Tab 키로 Drawer 내부만 순회)
- 닫힘 시 트리거 요소로 포커스 복귀

## 4. 도움말 (`/help`)

### 4.1 도움말 목록 페이지

**Header:** ← 뒤로가기 + "도움말" 타이틀

**콘텐츠:** 가이드 카드 목록

| # | 제목 | 설명 | 썸네일 |
|---|------|------|--------|
| 1 | 창 유형 알아보기 | 확장/비확장 발코니 창의 차이점 | 이미지 |
| 2 | 사이즈 측정하는 법 | 안쪽 치수를 정확히 재는 방법 | 이미지 |
| 3 | 브랜드별 특징 | LX, KCC, 청암의 장단점 비교 | 이미지 |
| 4 | 견적부터 시공까지 | 전체 프로세스 안내 | 이미지 |

**카드 UI:**
- 썸네일 (좌측 또는 상단) + 제목 + 설명 (2줄 말줄임)
- 클릭 시 `/help/:id`로 이동

### 4.2 도움말 상세 페이지 (`/help/:id`)

**Header:** ← 뒤로가기 + "도움말" 타이틀

**콘텐츠:**
- 상단 이미지 (전체 너비)
- 제목 (h2)
- 본문 (Markdown 렌더링 또는 Rich Text)
- 관련 도움말 링크 (하단)

## 5. 고객센터 (`/contact`)

### 5.1 문의 폼

**Header:** ← 뒤로가기 + "고객센터" 타이틀

| 필드 | 타입 | Placeholder | Validation |
|------|------|-------------|------------|
| 이름 | text input | 이름을 입력해주세요 | 2~20자, 한글/영문만 (필수) |
| 이메일 | email input | example@email.com | 이메일 형식 검증 (필수) |
| 전화번호 | tel input | 010-0000-0000 | 010 시작, 10~11자리 (필수) |
| 문의 내용 | textarea | 문의하실 내용을 입력해주세요 | 최소 10자, 최대 1000자 (필수) |

**버튼:** "문의하기" — Primary Button

### 5.2 Validation 상태

| 상태 | UI |
|------|-----|
| 기본 | 기본 border (`#E5E5E5`) |
| 포커스 | Primary Color border |
| 에러 | Red border + 하단 에러 메시지 (빨간색, caption) |
| 완료 | Green checkmark 아이콘 |

### 5.3 에러 메시지

| 필드 | 에러 조건 | 메시지 |
|------|-----------|--------|
| 이름 | 미입력 | "이름을 입력해주세요" |
| 이름 | 2자 미만 | "이름은 2자 이상 입력해주세요" |
| 이메일 | 미입력 | "이메일을 입력해주세요" |
| 이메일 | 형식 오류 | "올바른 이메일 형식을 입력해주세요" |
| 전화번호 | 미입력 | "전화번호를 입력해주세요" |
| 전화번호 | 형식 오류 | "올바른 전화번호를 입력해주세요" |
| 문의 내용 | 미입력 | "문의 내용을 입력해주세요" |
| 문의 내용 | 10자 미만 | "최소 10자 이상 입력해주세요" |

### 5.4 제출 동작

1. 유효성 검사 통과 시 API 호출 (`POST /api/contact`)
2. 성공 시: "문의가 접수되었습니다" 토스트/모달 표시 → 홈으로 이동
3. 실패 시: "문의 접수에 실패했습니다. 다시 시도해주세요" 에러 메시지
4. 접수된 문의는 **admin@ezpzspace.com**으로 이메일 발송
5. 사용자에게 접수 확인 **이메일 + SMS** 발송

### 5.5 API

```
POST /api/contact
Content-Type: application/json

{
  "name": "홍길동",
  "email": "user@example.com",
  "phone": "01012345678",
  "content": "문의 내용..."
}

Response 200:
{
  "success": true,
  "message": "문의가 접수되었습니다"
}
```

## 6. 공통 UI 명세

### 6.1 색상

| 토큰 | 값 | 용도 |
|------|----|------|
| Primary | `#2563EB` (Blue 600) | CTA 버튼, 강조 |
| Primary Dark | `#1D4ED8` (Blue 700) | 호버 상태 |
| Gray 100 | `#F5F5F5` | 배경 |
| Gray 300 | `#D4D4D4` | Border |
| Gray 500 | `#737373` | 보조 텍스트 |
| Gray 900 | `#171717` | 본문 텍스트 |
| Error | `#DC2626` (Red 600) | 에러 상태 |
| Success | `#16A34A` (Green 600) | 성공 상태 |

### 6.2 타이포그래피

| 토큰 | 크기 | 굵기 | 행간 |
|------|------|------|------|
| h1 | 28px | Bold (700) | 1.3 |
| h2 | 24px | Bold (700) | 1.3 |
| h3 | 20px | SemiBold (600) | 1.4 |
| body1 | 16px | Regular (400) | 1.5 |
| body2 | 14px | Regular (400) | 1.5 |
| caption | 12px | Regular (400) | 1.4 |

### 6.3 반응형 Breakpoint

| 이름 | 범위 | 설명 |
|------|------|------|
| Mobile | < 768px | 기본 (모바일 퍼스트) |
| Tablet | 768px ~ 1023px | 중간 |
| Desktop | ≥ 1024px | 최대 너비 1200px, 중앙 정렬 |

## 7. 비기능 요구사항

- **성능**: LCP < 2.5초, FCP < 1.8초
- **SEO**: 메타 태그, Open Graph, 시맨틱 HTML
- **접근성**: WCAG 2.1 AA 준수
- **분석**: 페이지뷰, CTA 클릭 이벤트 트래킹
